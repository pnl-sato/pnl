## 2. 業務概要

クライアント企業から依頼されたポジションに対し、候補者をマッチングし選考プロセスを進めるヘッドハンティング業務。
主要な成果物は以下：

- **スカウト文**（候補者へ向けた一次接触メッセージ）
- **推薦文／推薦状**（クライアント企業へ候補者を紹介する文書）
- **クライアントとのメール／打ち合わせ文章**（要件確認・進捗報告・日程調整）
- **面談メモ・打ち合わせメモ**（候補者・クライアント双方）

## 3. ツール構成と役割分担

| ツール | 役割 |
|---|---|
| **Notion** | 業務DBの正本（企業・ポジション・候補者・パイプライン・選考進捗・KPI） |
| **Craft** | 文章作成の作業場（推薦状・スカウト文の下書き、面談メモ、AIプロンプト管理） |
| **Gmail** | クライアント／候補者とのメール |
| **Slack** | 社内コミュニケーション |
| **Salesforce** | CRM（一部の管理） |
| **NotebookLM** | 打ち合わせ・面談ログの要約 |
| **ChatGPT / Gemini** | プロンプト実行（Craft内にプロンプト保管） |

Notion の DB 構造詳細は `notion_structure.md` を参照。

## 6. 動作の原則

- 候補者名・企業名・ポジション名など固有情報は、ユーザーから明示された値のみを使う。推測で埋めない。
- 推薦文・スカウト文の文字数指定（例：280-300字）は厳守する。
- 出力は Craft のマークダウン記法に貼り付ける前提でフォーマットする。
- 候補者の経歴・面談メモが添付されている場合、その内容を踏まえてから書く。情報不足時は何を確認したいか質問する。

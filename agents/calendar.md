# 面接・面談カレンダー登録エージェント

候補者の**選考プロセス上の面接（クライアント企業との面接）**の確定予定を Google Calendar に登録する際の規則。
「面接（面談）を登録して」「カレンダーに入れて」と依頼されたら、本書に沿って必ず同じ形式で登録する（2026-06 佐藤指示）。

---

## 0. 適用範囲（2つのカレンダー登録ルールの棲み分け）

カレンダー登録には用途別に2つのルールがある。取り違えない。

| シーン | 形式・ルール | 参照先 |
|---|---|---|
| **佐藤が主催する候補者インタビュー**（一次の P&L 面談・カジュアル面談で佐藤が Meet を発行して話すもの） | `［Online:{分}］インタビュー (氏名)`、Google Meet を自前付与、候補者を attendee に追加 | `agents/candidate-mail.md`「カレンダー登録フォーマット（候補者面談・全件共通）」 |
| **選考プロセス上のクライアント面接**（候補者が企業と受ける面接。佐藤は同席 or 不参加でトラッキング） | `{種別}：{企業}｜{候補者}`、`03_面接・面談` カレンダー、先方発行 Meet を使用、attendee なし | **本書** |

迷ったら「Meet を佐藤が発行して佐藤が話す＝candidate-mail 側」「企業の選考枠＝本書」で判断する。

> **日程調整リンク（Google 予約ページ）の正本：** こちらから空き枠を提示する Google カレンダー予約ページ（60/45/30分）は、**候補者面談用＝`agents/candidate-mail.md` §12**、**企業打ち合わせ用＝`agents/client-writing.md`「日程調整」**にまとめてある。所要時間は相手の温度感で選ぶ（前のめり→長め／ガード気味・初回・多忙→短め）。

## 1. 登録先カレンダー（最重要）

- **必ず `03_面接・面談` カレンダーに登録する。プライマリ（`sato-y@pnl.co.jp`）には入れない。**
- Calendar ID: `c_a22e56412713edf785d7cfc3f4c2c23d829b967c7bfe9ead7549b6c8c9d4ca79@group.calendar.google.com`
- `mcp__Google_Calendar__create_event` の `calendarId` に上記を必ず指定する（省略するとプライマリに入ってしまう）。

## 2. タイトル形式

```
{種別}：{企業名}｜{候補者氏名}
```

- 区切りは全角コロン `：` と全角縦棒 `｜`。
- **種別**の例：`カジュアル面談` / `1次面接` / `2次面接` / `3次面接` / `最終面接` / `オファー面談`。クライアントが「面接」と呼ぶか「面談」と呼ぶかは相手の用語に合わせる。
- 1つの選考フェーズが**複数面接官で枠が分かれる**場合（例：2次が篠塚・影澤の各1時間）は、末尾に `（{面接官}／{役職}）` を補って区別する。
  - 例：`2次面接：ギフティ｜八木 俊輔（篠塚／事業本部長）`
- 実在例：`カジュアル面談：ギフティ｜八木 俊輔` / `1次面接：メドレー｜有光 貴裕`

## 3. 日時

- JST（`timeZone: Asia/Tokyo`）で開始〜終了を入れる。通常は所要1時間。
- ATS／クライアントが提示した確定枠の時刻をそのまま使う。

## 4. 説明（description）

簡潔に以下を含める（分かる範囲で。無いものは無理に埋めない）：

- 面接官・役職（例：篠塚さん＝事業本部長／次期事業会社代表）
- 実施方法：オンライン（Google Meet）／対面（場所）
- 佐藤の同席有無（同席する場合はカメラ・マイクOFF 等の条件も）
- Meet URL：ATS／クライアントの確定メールに記載のものを使う。**未受領なら「Meet URL は先方確定メール受領後に追記」と明記**し、届いたら本イベントに追記する
- 関連メモ（同フェーズの別面接が調整中、など）

## 5. 参加者・通知・Meet URL の扱い

- **候補者・クライアントを attendee に入れない**（招待メールが飛んでしまうため）。本カレンダーは佐藤の自己管理用。
- Google Meet URL は**先方（ATS／クライアント）が発行したものを使う**。`addGoogleMeetUrl` での自前生成はしない。確定メールで受領したら description に追記（必要なら `googleMeetUrl` で添付）。

## 6. 登録タイミング

- **確定した枠のみ登録する。** 候補者が枠を選び、ATS で確定したものだけ。仮の候補日程・調整中のものは登録しない。
- 同フェーズで一部のみ確定（例：篠塚は確定・影澤は調整中）の場合は、確定分だけ登録し、未確定分は description に「調整中」と注記しておく。

## 7. 手順

1. Calendar ID は本書の値を使う。形式に迷ったら `mcp__Google_Calendar__list_events`（calendarId に `03_面接・面談`）で既存イベントを確認。
2. `mcp__Google_Calendar__create_event` で calendarId・summary・startTime・endTime・timeZone・description を指定して作成。
3. 作成後、イベントリンクと内容をユーザーへ報告。Meet URL 未受領ならその旨と「届いたら追記する」を添える。

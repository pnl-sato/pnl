## 1. 面談の基本スタンス（佐藤）

上長から繰り返し評価されている佐藤の動き。

- **キャリア面談型**：案件ありきではなく、まず本人のキャリア方向性・志向を聞いてから提案先を組み立てる。
- **対面・会食を重視**：選考結果に依存せず食事をセッティングして本音を引き出す。特に大沢さん（メドレー結果待ち時のGW会食）、小林さん・坂美さん（GW中ランチで集中提案）の事例で評価されている。
- **フック案件 → 横展開**：魅力ある案件（例：ブルートープ＝クラフトビール、GTT＝セキュリティ、メドレー）で接点を作り、ギフティ／バイセル等の注力案件へ繋ぐ。
- **現実認識を促す**：希望条件と市場のギャップを意図的に経験させる。例：「現年収維持 × 小規模アーリー」を打診し続けて本人が条件のトレードオフに気づくまで待つ（小林さんへの打ち止めアプローチ）。

## 2. ヒアリングの原則（上長フィードバック由来）

### 2-1. 言葉の裏のニーズを推測する

候補者の発言を額面通りに受け取らず、その裏にある真のニーズを言語化する。

| 候補者の発言 | 裏にある可能性のあるニーズ |
|---|---|
| 「スピード感が合わない」 | 裁量権が欲しい（井上さんの例：ビズリーチが意思決定権を渡さない環境への不満） |
| 「IT/SaaSはお腹いっぱい」 | 手触り感のあるリアルな事業に挑戦したい（眞島さんの例：農業・不動産・オールドインダストリーへの志向） |
| 「キャリアを解放してくれる場所が良い」 | 専門性で縛られず、自分が呼ばれていると実感できる場 |
| 「海外買収PMI経験者がほしい」（クライアント発言） | 実際は「若くて地頭が良く、素直な人」がほしい（ブルートープの例） |

### 2-2. ジョブ型／ジェネラリスト型の見極め

- **ジョブ型**（明確な専門性あり）：JD の記載要件で直接語れる。
- **ジェネラリスト型**：JD を見て「自分が呼ばれている／自分に合う」と実感できるかが重要。
  - 例：大沢さん（メドレー選考お見送り後）には、カケハシの「グループ横断戦略設計推進リーダー」「事業変革DX推進シニアマネージャー」のような書き方の案件をぶつける。

### 2-3. 「事業家」マインドの見抜き方

レジュメから「人としての想像力」を読み解く。

- 大手出身でも「与えられた棚割りをこなすだけの営業／マーケ」は経営層には届かない。
- 「軽井沢蒸留所の立ち上げ」「PWC出身で蒸留所立ち上げ」のような泥臭く事業を作った経験者は高評価。
- ECプラットフォームの事業成長、商品企画など「事業を動かす」経験があるか。

### 2-4. ベンチャー適性の見極め

- ベンチャーは完全なジョブ型採用ではなく「みんなで一緒に文化祭ができるか」というカルチャー重視。
- 結局採用されるのは「若くて地頭が良く、素直」な人。
- ベンチャー向けターゲットは30代中盤までが中心。
- 例：バイセルの事業企画は30代前半が中心 → 40代候補者の通過は厳しい。

## 3. 面談の進行モデル

### Phase 1：オープニング・関係構築（5-10分）

- フック案件（ブルートープ・メドレー等）で接点を作った場合、その関心点に触れる。
- 食事の場であれば、まず雑談から入る。
- 「今日はキャリアを一緒に整理する場」と位置づける（売り込みのスタンスを最初に出さない）。

### Phase 2：志向ヒアリング（20-30分）

- 現職での具体的な不足／充足を聞く。
- 抽象語（「裁量」「スピード感」「成長」「フェーズ」など）が出たら**必ず深掘り**。「具体的にはどういう瞬間にそれを感じますか？」
- 過去の意思決定パターン（前職→現職の動機、退職理由の真相）。
- 「やりたくないこと」も聞く（年収レンジ、フェーズ、領域、働き方、地域）。
- 家族・健康・働き方の制約（リモート希望、転居可否、出社可能日数）。
  - 例：井上さん＝奥様難病で週2-3日リモート希望、岩本さん＝海外駐在ではなく海外出張希望。
- 転職時期の目安（「11月頃」「秋から冬」など曖昧でも引き出す）。

### Phase 3：仮説提示・反応観察（10-20分）

- ヒアリングを踏まえ、その場で **2-3社の候補を仮置き**する（1社だけは避ける、選択肢があると意思決定の手応えが出る）。
- 各社の「面白さ」と「向いていなさそうな点」を両方フェアに伝える。
- 候補者の表情・質問の方向で志向の解像度を上げる。
- 「これは違うかも」というネガティブ反応も歓迎して、次の角度を提示する。

### Phase 4：次アクション設計（5分）

- 具体的に次にどの案件をいつまでに整理するか。
- 候補者側のスケジュール感とすり合わせる。
- 必要なら次回面談・会食の日程を仮押さえ。
- ヘッドハンター的アプローチを取る場合は、企業側へ持ち込む可否を本人に確認。

## 4. トークスクリプト作成時のチェックリスト

- [ ] フック案件の関心点を冒頭で確認しているか
- [ ] 抽象語（「スピード感」「裁量」「成長」など）には必ず深掘り質問を置いているか
- [ ] 候補者がジョブ型／ジェネラリスト型のどちらかを見立てて、提案先の角度を変えているか
- [ ] 現実認識を促すフェーズ（条件のトレードオフ）を意図的に組んでいるか
- [ ] 提案案件は最初に3社程度を提示する設計か
- [ ] 候補者の発言の裏のニーズを言語化する問いがあるか
- [ ] 「やりたくないこと」「働き方の制約」を聞く設計があるか
- [ ] 次アクションが具体的に決まる構成か（曖昧な「またご連絡します」で終わらない）

## 5. やってはいけない面談スタンス

- ❌ 案件ありきで本人の志向確認を後回しにする
- ❌ 候補者の発言を額面通りに受け止めて深掘りしない（「スピード感」「成長」などをそのまま記録する）
- ❌ 1社しか提案しない
- ❌ 年収・フェーズの矛盾をスルーして提案する
- ❌ ベンチャー候補者にジョブ型JDをそのまま提示する
- ❌ 大手出身者の経歴を「肩書きの良さ」だけで評価する（事業家マインドの有無を見る）
- ❌ 「あんま寝かせると取れなくなる」優秀層へのフォローを遅らせる

## 6. ヘッドハンター的アプローチ（優秀層向け運用）

求人に当てはめるのではなく、企業側に直接持ち込むアプローチ。

- **対象**：レジュメ・面談で自信を持って推せる層（岩田さん、恩田さん＝元リクルート／ウォンテッドリー取締役、眞島さん＝ストックマーク10年）。
- **持ち込み先**：解像度高く語れる企業（ヤマシタ、カケハシ、バイセル、テオリア、ブルード、アルバリンクなど）。
- **打診の型**：「最近会った中で一番良い人材がいる」と経営層に直接ピッチ → ポジションがなくても会いたいと思わせる。
- **面談時の追加ステップ**：このアプローチを取る場合、「具体的に企業に話を通してよいか」を本人に確認するパートを面談内に組み込む。

## 7. 注力フックと横展開先（直近の組み合わせ）

トークスクリプトを作る際、どの案件をフックに使い、どこへ横展開するかの参照表。

| フック案件 | 主な横展開先 | 想定候補者属性 |
|---|---|---|
| ブルートープ（クラフトビール） | ギフティ本体PdM、バイセル、テオリア、ガバテック、アルバリンク | 30代中盤までの事業立ち上げ経験者、PdM・Biz層 |
| GTT（セキュリティ） | DMM、メドレー、ヤマシタ、ギフティCISO候補 | セキュリティエンジニア、CISO候補、プレイヤー層 |
| メドレー（医療×IT） | バイセル、カケハシ、DMM | ジェネラリスト・キャリア解放志向 |
| リテーナー（ギフティ等） | アップフィー、本体PdM | 若手〜中堅 |

## 8. クライアント別の見立て（提案時に踏まえる）

| クライアント | 給与傾向 | 年齢層・特徴 |
|---|---|---|
| ギフティ | 基本スライド限界、強くない | リテーナーで集めた若手を横展開可 |
| メドレー | 取りに行くためしっかりオファー | 専門性ある層に強い |
| バイセル事業企画 | 中程度 | 30代前半中心、40代は通過しにくい |
| ブルード | 成長中・鉄則の提案先 | ベンチャーマインド層 |
| カケハシ | 中〜高 | 横断戦略・DX推進系のジェネラリスト案件あり |

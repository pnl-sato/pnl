スカウト返信後〜選考完了までに候補者へ送るメールのガイド。
（スカウト一次接触の文面は `agents/scout.md` を参照。本ファイルは「返信が来た後」「面談後」のメール体）

実サンプルは Gmail 送信箱（2024〜2026 年）から抽出。

### 書き出し（古市からの引き継ぎ案件）

```

よろしくお願いいたします。
```

### カレンダー登録フォーマット（候補者面談・全件共通）

候補者との面談（インタビュー）を Google カレンダーに登録する際は、**必ず以下の形式に統一する**（2026-06 佐藤指示。既存イベントの法則に準拠）。

> ※ これは**佐藤が主催する候補者インタビュー**（佐藤が Meet を発行して話す一次面談）の登録形式。候補者がクライアント企業と受ける**選考プロセス上の面接**は別ルール（`03_面接・面談` カレンダーに `{種別}：{企業}｜{候補者}` で登録、先方発行 Meet を使用、attendee なし）。そちらは `agents/calendar.md` を参照。

- **タイトル**：`［Online:{所要分}］インタビュー (候補者氏名)`
  - 角括弧は**全角**「［ ］」、その中は `Online:60` / `Online:30`（実際の所要時間の分数）。
  - 氏名は**半角括弧** `( )` で囲み、**敬称なし・姓名は連結**（例：`(成毛一行)` `(石川真衣)`）。
  - **漢字表記が確証できない候補者は、無理に漢字化せず本人の表記（ローマ字など）に合わせる**（例：LinkedIn 上の `Hiroyuki Takiyama` のみで漢字が裏取りできない場合は `(Hiroyuki Takiyama)`）。
  - 例：`［Online:60］インタビュー (成毛一行)` / `［Online:30］インタビュー (石川真衣)`
- **Google Meet を必ず付与する**（`create_event` の `addGoogleMeetUrl: true`）。発行された Meet URL を候補者への面談案内メッセージに記載する。
- **説明欄**（推奨）：案件（社名・ポジション）／媒体／当日の論点／事前確認事項などを箇条書きで残す。
- **参加者**：候補者のメールアドレスが分かる場合は attendee に追加（URL が自動共有される）。LinkedIn InMail など**メール不明の場合は Meet URL のみ作成し、本人へは別途メッセージで共有**する。
- 補足：候補者の自己予約ツール経由のイベントには、前後に30分の「作業」バッファ（前作業／後作業）が自動生成されることがある。手動登録時は必須ではない。

## 4. 面談後の御礼

### 件名
- `【面談の御礼】Pole&Lineの佐藤です`
- `【ご面談の御礼】Pole&Lineの佐藤です`

### 構造
```
{姓}様
Pole&Lineの佐藤です。

本日はお忙しい中、お時間いただきありがとうございました。
{相手の話で印象に残った点を具体的に1〜2文：「ヘルスケア領域でのご経験やキャリアに対するお考え、大変参考になりました。」「お話を伺いながら、{志向}を次のキャリアで重視されている点が、とても印象に残っています。」}

{次アクションの提示}：
- 紹介する案件のJDや参考情報を添付して送る
- 「お時間あるタイミングで、気になるところだけ拾い読みいただければ大丈夫です。」と負担を軽くする

引き続き{姓}さんのご状況やご志向に合うものがあればご連絡させていただければと思います。
今後もよろしくお願いいたします。

--
Pole&Line合同会社
佐藤 雄太｜SATO Yuta
Mobile：080-3930-1714
```

### ポイント
- 「印象に残った点」は必ず**具体的**に書く（「とても勉強になりました」のような空文句は使わない）
- 候補者の呼び方は本文中は「{姓}さん」もよく使う（御礼メールでの親しみ表現）

## 6. 書類依頼・確認

### 【マスト】書類依頼前のプライバシーポリシー同意確認

職務経歴書・履歴書など個人情報を含む書類を候補者に依頼する**前**に、SF Contact の **`PersonalInformation1__c`（個人情報の取扱い同意）** を確認する。

| 値 | 対応 |
|---|---|
| `同意書サイン済` または `不要` | 同意リンク添付は**不要**、通常の書類依頼でOK |
| `なし` または空 | **書類依頼メールに同意フォームURLを必ず併記**し、同意 → 書類提出の順で依頼する |

**同意フォームURL：** `https://docs.google.com/forms/d/e/1FAIpQLSf5VhQ8uZ_maUd6wsVy5_RKg6hgLgEE22wiuZvApADb8xOBEw/viewform`
（Craft 正本：「[プライバシーポリシーへの同意](https://docs.craft.do/) `887EB71A-FE4C-4B52-AA03-5BC507EC20F3`」）

同意フォームを送付した候補者は、本人が回答送信した時点で同意完了。回答受領後、SF 側の `PersonalInformation1__c` を `同意書サイン済` に更新する（自動連携あり／手動補正の必要があれば確認）。

#### 書類依頼メールへの組み込み例

```
書類のご共有にあたり、弊社の個人情報の取扱いについてご確認・ご同意のお願いです。
お手数ですが下記フォームにご回答のうえ、書類をお送りいただけますと幸いです。
（所要1〜2分／同意は1回限り、以降の書類依頼では不要となります）

https://docs.google.com/forms/d/e/1FAIpQLSf5VhQ8uZ_maUd6wsVy5_RKg6hgLgEE22wiuZvApADb8xOBEw/viewform
```

Facebook Messenger・LINE 等のチャットの場合も同様に、書類依頼と同じメッセージ内に同意フォームURLを含める。同意がチャネル外（フォーム）で完了するため、書類受領前に必ず分離しておく。

## 7. 選考結果通知（見送り）

### 構造
1. **結論を最初に明示**
2. **企業側の判断理由を率直に伝える**（候補者を否定しない表現で）
3. **謝罪は短く**
4. **次の話への布石**（あれば）

### 実例パターン1：書類見送り
```
{姓}様
Pole&Lineの佐藤です。

{社名}の件、お待たせしておりました。
ご検討いただいた結果、今回は書類の段階で見送りという判断となりました。

今回のポジションでは{企業が求めていた要素}がより強く期待されていたようで、その点で{姓}様が大切にされている{候補者の志向}とは、若干のズレが{あったようです／生まれてしまったようです}。

良いご縁をお作りできず、申し訳ございません。
{次の話があれば：別件で〜}
```

### 実例パターン2：面談後見送り
```
{社名}の件ですが、面談内容を踏まえて先方に確認したところ、{姓}さんの強みと先方の期待するところにギャップがあるとのご判断でした。
{具体的なギャップ}：「○○のご経験は魅力に映る一方で、△△における経験の厚みとビジネスモデルの違いが課題とのことです。」

良いご縁をお作りできず、申し訳ございません。
別件{で〜／また機会あればご相談させてください}
```

### ポイント
- 企業側の言葉として伝える（「先方のご判断」）→ エージェントの主観ではない
- ネガティブを和らげる：「ズレが生まれてしまったようです」「ギャップがある」
- 「良いご縁をお作りできず、申し訳ございません」は定番フレーズ

もしよければ近況含めてキャッチアップできればと思っていて、今度お茶かランチでもご一緒できればと考えていますが、ご都合いかがでしょうか？

### お久しぶり再接触＋案件teaser（社名伏せ）｜2026-06 実送信例

1年以上ご無沙汰の候補者へ、近況伺いと社名を伏せた案件提案を兼ねる軽いトーン。すぐの転職を前提とせず「久しぶりに話しませんか」で接点を取り戻す形。佐藤が実送信した文面（家庭事情に配慮しながら、過去面談での本人発言を引用して訴求）。

```
{姓}様

ご無沙汰しております。Pole&Lineの佐藤です。
昨年お話しして以来となってしまい失礼しております。
その後、{ご家族の近況を気遣う一文}。あわせて{姓}様も、{現職の状況}が一段落されていればと思いつつ、ふとどうされているかと気になっておりました。

以前お話しした際に、{本人が過去に語った志向・転職軸をそのまま引用}、とおっしゃっていたのが印象に残っております。
先日、その志向にかなり重なるのではと感じるポジションのご相談をいただきました。{社名・業界は伏せ、志向に重なる特徴だけを2〜3文：社会的に意義のある事業／次の世代に残る構造的な課題／変革の渦中にある成長企業 など}。
{役割の格を一言：経営の中枢に近い立場で〜をリードいただくお話}で、{姓}様のこれまでのご経験が活きる余地が大きいのではと感じております。

詳細は社名を含めてお話しした際にお伝えできればと思いますが、すぐのご転職を前提とせずとも、久しぶりに近況のキャッチアップも兼ねて一度オンラインでお話しできればうれしく思います。もしご都合よろしければ、30分ほどお時間をいただけないでしょうか。下記より、ご都合の良い日時をご選択いただけますと幸いです。

{日程調整リンク}

ご多用のところ恐縮ですが、ご検討いただけますと幸いです。
```

**このパターンの学び（佐藤の手直しから抽出）：**

- **再接触の冒頭は媒体を特定しない。** 「お電話でお話しして以来」ではなく **「昨年お話しして以来」** とシンプルに。電話・メール・面談のいずれだったかを限定せず、軽く触れる。
- **「お会いした際に」より「お話しした際に」。** 面談がオンライン主体のため、対面に限定する語（お会いした）を避け、オンライン/対面どちらも含む「お話しした」を使う。
- **本人の過去発言を引用して訴求する。** 過去面談・文字起こしにある本人の言葉（転職軸・志向）をそのまま引くと、「覚えてくれている」という信頼が効く。引用は脚色せず本人の語り口に寄せる。
- **社名・業界・数値・年収は本文に出さない。** 志向に重なる特徴（社会的意義／構造的課題／変革フェーズ／経営直下の格）だけで引きを作り、詳細は「お話しした際に」と面談に送る。
- **「すぐの転職を前提とせず」と必ず逃げを作る。** 家庭事情等で温度が読めない相手には、近況キャッチアップを主目的に置き、案件はあくまで副次。プレッシャーを与えない。
- **家族・健康への気遣いは病名等の機微に触れず一段やわらかく。** 「その後ご体調はいかがでしょうか」程度に留め、具体の病状には踏み込まない。

## 11. 共通ルール

- 件名は `Pole&Line` を含める（社内識別のため）
- 候補者宛の本文中での呼び方は **「{姓}様」**（堅め）または **「{姓}さん」**（親密な関係・面談後）を使い分け
- **宛名の漢字を推測で当てない（2026-06 佐藤指示）。** LinkedIn など媒体IDがローマ字／英字表記で、漢字氏名が正本（Notion/SF/Craft/過去やり取り）で裏取りできない場合は、**勝手に漢字化せず媒体IDの表記をそのまま使う**（例：媒体ID `Kohei Tomoda` → 宛名 `Tomoda様`。given-family 順なので姓＝`Tomoda`）。誤った漢字（同姓異字など）を当てるリスクを避けるため。漢字が正本で確認できる候補者のみ `{漢字姓}様` を使う。カレンダー登録の氏名表記も同じ原則（§2「カレンダー登録フォーマット」参照）。
- **署名欄は付けない（メール＝Gmail の場合）**（2026-06 佐藤指示）。Gmail は送信時に署名が自動付与されるため、Claude が作成するメール下書きは本文で終える。参考までに正式フル署名は以下：
  ```
  --
  Pole&Line合同会社
  佐藤 雄太｜SATO Yuta
  Mobile：080-3930-1714
  Mail：sato-y@pnl.co.jp
  Website：https://pnl.co.jp
  ```
- **LinkedIn は「初回メッセージ以外」に軽い署名を付ける（2026-06 佐藤指示）。** LinkedIn は**初回メッセージにのみ署名が自動付与され、2通目以降は自動で入らない**。そのため、スカウト返信など**初回以外の LinkedIn メッセージ**は、本文の結び（「ご返信お待ちしております。」等）のあとに軽い署名を手で付ける：
  ```
  Pole&Line
  佐藤
  ```
  フル署名（住所・電話・URL 入り）は不要。LinkedIn の**初回**メッセージ（最初のスカウト等）は自動署名が入るので付けない。
- 古市と佐藤の役割分担を明示する（「弊社の古市から〜」「当日はわたくし佐藤が〜」）
- 「お忙しい中」「お時間頂き」など相手への配慮表現を入れる
- 結果通知や調整失敗は**短く謝り、原因を率直に伝える**。長文の言い訳はしない。
- **候補者に日程（空き枠）の提示を依頼するときは、候補者を「待たせている側」にしない**（2026-06 佐藤添削）。「先方がお待ちです」はプレッシャーや遅延責任を負わせる印象になるため避ける。代わりに「（私から先方へ）{姓}様から日時提示いただき調整する旨でお伝えしておりますので、お手隙の際に来週〜再来週で候補をいくつか幅をもってお知らせください」のように、**調整の主導はエージェントが引き取り**、候補者には低圧で幅をもらう。依頼の枕詞も「お手数ですが」より**「お手隙の際に」**が当たりが柔らかい。多忙なハイレイヤー候補ほど有効。
- **強調副詞（強く／非常に／もっとも 等）は初稿から抑える**。控えめな断定にする（2026-06 佐藤添削より。「強く共感」→「共感」、「もっとも近い」→「近い」）。
- **箇条書きは1項目1メッセージ**。見出し（「◯◯候補」等）で述べた役割を本文で言い直さない（冗長な役割再掲はカット）。
- **日付語（本日／昨日／先日）は送信タイミングと事象日を `Today's date` で検算してから書く**（面談翌日に送るなら「昨日は」）。
- **既知の相手への継続連絡では冒頭挨拶を最小化する**（「お世話になっております。」を重ねず「Pole&Lineの佐藤です。」程度に）。初回接触の丁寧な定型とは使い分ける。

## 12. 日程調整リンク（Google カレンダー予約ページ｜候補者面談用）

候補者宛メッセージ（面談案内・スカウト返信・再コンタクト等）で `{日程調整リンク}` を入れるときは、以下の佐藤の予約ページを所要時間別に使い分ける（2026-06 佐藤共有）。**企業打ち合わせ用は別リンク**（`agents/client-writing.md` を参照）。取り違えない。

> **用語の区別（2026-06 佐藤指示・重要）：** ここでいう「候補者面談」は**佐藤と候補者の面談＝「P&L面談」**を指す（佐藤が Meet を発行して話す一次面談。スカウト返信後の最初の接点）。**「カジュアル面談」は企業と候補者の面談**を指す別物で、選考プロセス上のステップ（`agents/calendar.md` のクライアント面接側）。スカウト返信への打診や記録で、佐藤との30/45/60分面談を「カジュアル面談」と書かないこと（正：P&L面談）。Notion パイプライン選考状況・SF phase でも、佐藤との面談は「P&面談調整中／P&L面談」「返信あり、P&L面談日程調整」を使う。

| 所要 | 予約リンク |
|---|---|
| 60分 | https://calendar.app.google/sbAiGmW8ev6Y7b1w6 |
| 45分 | https://calendar.app.google/gFyXmjZaoe3LBmtNA |
| 30分 | https://calendar.app.google/GxNMFrrRyukn5Q3P8 |

**時間の選び方（2026-06 佐藤指示）：** **できれば60分で確保したい**が、機械的に60分を貼らず、**候補者側の温度感を推測して接点を作る上で適切な長さ**を選ぶ。

- **60分**：転職意欲が顕在化している／じっくり話したい志向／こちらから深く擦り合わせたい本命候補。
- **45分**：関心はあるが温度が読みきれない中間。標準的な初回カジュアル面談。
- **30分**：多忙なハイレイヤー、ガード気味（社名確認だけ・返信が短い等）、まずは接点づくり優先の初回。長い枠は心理的ハードルになるため、短く始めて次につなげる。

迷ったら「相手の前のめり度が高い→長め／ガード気味・初回・多忙→短め」で判断する。

## 13. iMessage / チャットでの軽い連携（日程・URL共有）

面識のある候補者（特に関係性が近い相手）への iMessage / SMS / チャットでの日程・URL共有は、**メールより大幅に削ぎ落とす**。2026-06 佐藤の手直しから抽出。

実送信例（クライアントとのオンライン日程確定を候補者へ連携）：
```
河田さんとのオンライン、6/22（月）16:00〜17:00で確定しました！
Meet URL：https://meet.google.com/xxx-xxxx-xxx

前回お話しした通り、条件というよりお互いの温度感とタイミングをすり合わせる場、という位置づけです。ご確認お願いします。
```

**学び（佐藤の手直しから抽出）：**

- **冒頭の宛名（「{姓}さん、」）は付けない。** 1対1のチャットでは相手が自明なので省く。メール体との最大の差。
- **付帯情報を削る。** ダイヤルイン番号・PIN・タイムゾーン等は載せず、**Meet URL 一本に絞る**（必要なら別途）。チャットは要点のみ。
- **締めは「ご確認お願いします。」でシンプルに。** 「気になる点があれば事前にお気軽にご連絡くださいね」のような柔らかい誘い文は冗長。チャットでは短い依頼形で締める。
- **締めで過剰なサービスを申し出ない**（2026-06 佐藤添削）。「いつでも壁打ちお付き合いします」のような過剰な申し出は避け、「引き続きよろしくお願いします！」など自然な温度の一言で締める（継続関係のLINE/チャットは「！」で温度を合わせてよい）。
- **構成は「いつ・どこ（URL）」＋「その場の意味（位置づけ1行）」だけ。** メールの定型（件名・日時ブロック・結び）は持ち込まない。要点1〜2行＋位置づけ1行で完結させる。
- **iMessage とメールを両方送る場合、メールも「記録用のリーン版」にする**（2026-06 実例）。役割分担は iMessage＝主・即時連携／メール＝「改めてこちらでも」の記録コピー。**メール側は、ダイヤルインや位置づけの説明（温度感うんぬん）を重複するので省く**（位置づけはチャットで既に伝わっている）。冒頭で先のメッセージに触れ、日時・形式・URL の最小ブロックだけ残し、締めも短くする。署名なし。
  ```
  鳶本様

  先ほどお送りした河田さんとのオンライン面談の件、改めてこちらでもご連絡します。

  日時：2026年6月22日（月）16:00〜17:00
  形式：オンライン（Google Meet）
  URL：https://meet.google.com/xxx-xxxx-xxx

  以上、当日はよろしくお願いします。
  ```

### 運用情報リッチなチャット連携（面談確定の当日案内一式）｜2026-06 実送信例

クライアント面談（カジュアル面談）確定を候補者へ Facebook Messenger で連携した例（JMDC／井上様）。先方メールにあった**当日運用の情報一式**（接続トラブル時の対処・緊急連絡先・事前アンケート・文字起こしの可否・参考リンク）は漏らさず渡す一方、Claude 初稿にあった**冒頭の宛名・挨拶・自己紹介と、結びの長い気遣いを佐藤が削った**。§13 冒頭の単純な URL 共有（付帯情報を削る）とは「何を残し何を削るか」が違うので切り分けて記録する。

**学び（Claude 初稿 → 佐藤送信版）：**

- **冒頭は宛名も挨拶も自己紹介も付けない。** 「井上様／お世話になっております。Pole&Lineの佐藤です。」を全削除し、いきなり用件（「JMDCのカジュアル面談につきまして、日程が確定しましたのでご連絡します。」）から入る。継続中の1対1チャットでは送り手も相手も自明（§13冒頭の「宛名を付けない」を、挨拶・自己紹介の省略まで拡張）。
- **冗長な主語・修飾を削る。** 「先方と日程が確定」→「日程が確定」。
- **結びは「以上、ご確認お願いします。」で締める。** Claude 初稿の「万一10時が難しければすぐご連絡ください」「壁打ちが必要であればいつでもご一緒します」「どうぞよろしくお願いいたします」のような気遣い・申し出の重ねは、情報伝達が主のメッセージでは削る。
- **ただし"運用情報の payload"は削らない（§13「付帯情報を削る」との切り分け）。** §13 の単純な日時・URL 共有では Meet URL 一本に絞るが、本ケースのように**当日の段取り一式を渡すのが目的**のときは、接続トラブル時の対処・緊急連絡先（メール＋電話＋5分ルール）・事前アンケートURL・文字起こしの opt-out・参考リンクまで**先方メールの内容を漏らさず**残す。削るのは社交フレーム（宛名・挨拶・自己紹介・結びの重ね・冗長主語）だけで、候補者が当日必要とする実務情報は full に保つ。
- **緊急連絡先には佐藤への一報導線を残してよい。** 先方の緊急連絡先（interview-sos／代表電話＋5分ルール）に加えた「あわせて私（佐藤）にもご一報いただければ、こちらからも先方へ連携します」は、エージェントのフォロー線として送信版でも維持された（削らず残す申し出）。

## 14. 候補者からの辞退・転職活動クローズ連絡への受け返し（2026-06 佐藤実送信）

候補者から「現職に残る」「転職活動を一旦やめる」「今回は辞退する」等の離脱連絡が来たときの受け返し。関係を良好に保ったまま、相手の決定をそのまま尊重し、再接触の余地だけ低圧で残す。

実送信例（熊谷様｜現職残留＋担当事業/エリア変更で再チャレンジの連絡を受けて）：

```
熊谷様

Pole&Lineの佐藤です。
ご連絡いただきありがとうございます。

ご状況について、承知いたしました。

こちらこそ、また良いご縁やタイミングがありましたら、いつでもお声がけいただけますと幸いです。
その折にはあらためてお力になれればと思っております。

引き続きよろしくお願いいたします。
```

**学び（Claude 初稿 → 佐藤送信版）：**

- **相手の決定を解釈・評価して褒め返さない。** Claude 初稿の「納得感を持って前向きな決断をされたのだと拝察しております」「新たなフィールドでのご活躍を楽しみにしております」は、相手の内情を推し量る＋過剰な肯定で、離脱連絡への返信には重い。佐藤は **「ご状況について、承知いたしました。」の一文だけ**で受け止めた。こちらの解釈・感想を載せず、相手の決定をそのまま尊重するのがクリーン。
- **再接触の布石は残してよい。** 「また良いご縁やタイミングがありましたら、いつでもお声がけいただけますと幸いです」「その折にはあらためてお力になれればと思っております」は、プレッシャーなく関係を継続できる定番の結び。離脱＝終わりではなく、ボールを相手に置いて自然に閉じる。
- **重ね言葉を削る。** 「引き続きどうぞよろしくお願いいたします」→「引き続きよろしくお願いいたします」（「どうぞ」を削る）。
- **署名なし（§11）。** 既知の相手なので冒頭挨拶も「Pole&Lineの佐藤です。」程度に最小化する。

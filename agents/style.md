# 文体ガイド（共通）

佐藤 雄太の実際の Gmail / Craft / Slack のサンプルから抽出した文体ルール。
推薦文・スカウト文・クライアント連絡など、どの場面でも基本ルールはここに従う。

---

## 1. 基本トーン

- **ですます調・敬体**。
- 「優秀なビジネスマンとしての振る舞い」を意識。
- 角が立たない／ネガティブな印象にならない／過度にへりくだらない。
- 流れる文章を基本としつつ、要点は3つ前後の箇条書きで補強。
- 1文は短く、改行を活用して読みやすく。

## 2. 敬称・呼称

| 対象 | 呼称 |
|---|---|
| 候補者（本人宛） | `{姓}様` |
| 候補者（第三者への言及） | `{姓}様` / `{姓 名}様` |
| クライアント企業（宛先） | `貴社` / `〇〇株式会社御中` |
| クライアント担当者 | `{姓}様` |
| 自社名 | `Pole&Line` / `Pole&Line合同会社` |
| 自身 | `Pole&Lineの佐藤` / `佐藤 雄太` |

## 3. 頻出フレーズ

### 書き出し
- 「お世話になっております。Pole&Lineの佐藤です。」
- 「はじめまして。Pole&Lineの佐藤と申します。」
- 「ご覧頂き有難うございます。」（スカウト返信時など）

### 連絡理由
- 「〜について、要件の確認でご連絡いたしました。」
- 「ご経歴を拝見し、ご活躍いただけるのではと思い、ご連絡差し上げました。」
- 「ぜひ一度お打ち合わせのお時間を頂戴したくご連絡しました。」

### 提案・推し
- 「〜という印象を持っております。」
- 「〜とお見受けしており、貴社にマッチするのではと考えております。」
- 「〜にご関心をお持ちです。」
- 「短いお時間でも結構ですので、一度オンラインで詳細をご説明できればと思います。」

### 結び
- 「どうぞよろしくお願いいたします。」
- 「引き続きよろしくお願いいたします。」
- 「ご検討いただけますと幸いです。」

### 候補者の転職理由を伝えるとき（必ずポジティブに）
- 「現職への不満はなく、〜のフェーズを志向。」
- 「より責任範囲が大きいところに挑戦したい」
- 「次のステージに向けて、現職の中ではなく外部で挑戦したい」

## 5. やってはいけない表現

- ❌ 候補者の前職を否定的に書く（「組織が崩壊して」など）
  → ✅「組織の変化を経て、自身の経験が活きる環境を再考している」
- ❌ クライアントを上から目線で評価する
- ❌ 「絶対」「必ず」など断定的・過剰な表現
- ❌ カタカナ語の乱用（必要なものは使う）
- ❌ 候補者名を文字数カウントに含める（推薦文の300字制限など）
- ❌ 地の文での強調カギカッコ「」の多用
  → 地の文の流れを重視する。強調したい語があっても、カギカッコで囲まず自然に読ませる。
  → 例：❌「事業を作る」スタンス → ✅ 事業を作るスタンス
- ❌ 本人の発言をそのままカギカッコで断定的に引用する
  → 「ぐらい」「みたいに」「とおっしゃっていた」のようにソフトに引用し、本人にプレッシャーを与えない。
  → 例：❌「LINEギフトの責任者をやりたい」とお話しくださった
  → ✅ LINEギフトの責任者をやりたいぐらい、とお話しくださった

## 6. 表記ルール

- **アルファベット固有名と日本語の間に半角スペースを入れない**：まとまり感を優先。
  - ❌ DeNA 時代 → ✅ DeNA時代
  - ❌ Pole&Line の佐藤 → ✅ Pole&Lineの佐藤
- 読点は流れを大事に。テンポを作るためにあえて読点を連続させるのは可（例：「、とお話しくださった、その温度感が…」）。

## 7. フォーマット規約

- Craft に貼り付ける前提で、マークダウン記法で出力する。
- 見出しは `##` / `###` を用い、過度なネストは避ける。
- 箇条書きは `-` を使う。
- 強調は `**太字**` を控えめに。

## 8. Slack／LINE（同期型コミュニケーション）運用

メール（推薦文・要件確認など）よりも軽い温度感が基本。継続会話／既に関係性ができている前提のチャネル。

### 共通ルール
- 挨拶・名乗りは省略する（メールの「お世話になっております。Pole&Lineの佐藤です。」は不要）。
- 既に話している相手に対する促しの一言（「ご確認いただけますと幸いです」「お手数ですが」など）は基本不要。事実報告だけで十分。
- 絵文字は控えめ。`！` で温度感を作る。スタンプは別途併用可。
- 複数チャネル並行で連絡している場合は、片方に他方への共有完了を簡潔に明記する。
  - 例：「HERPの方にも一本お送りしました！（こちらの件もHERPで連携済み）」

### 表記（メールでも共通）
- 候補者・社外関係者への第三者言及時も `{姓}様` で統一。社内 Slack でも同じ。
  - ❌「鈴木さん」「鈴木代表」 → ✅「鈴木様」
- **候補者の発言を直接引用しない／そのまま流さない**。本人にプレッシャーをかけるリスクを避けるため抽象化して伝える。
  - 例：候補者発言「LINEギフトの責任者やりたい」→ クライアント側への伝達は「ギフト事業への関心が強い」
- 固有名詞は補足を一段添えると相手の理解負荷を下げる。
  - 例：「事業会社」→「事業会社（ギフティ社）」
- 解説的な情報は削る。相手も承知している前提情報を添えると押し付けがましくなる。
  - ❌「事業会社サイドの篠塚様」 → ✅「篠塚様」

### 経緯説明・お詫び
- 「経緯」を書くなら**相手にとって必要な情報だけ**に絞る。社内事情や認識ズレの詳細は不要なことが多い。
- HERP など先方の社内システム経由のメッセージでは、ややフォーマル目に整えつつも、5/28ランチや認識ズレなどの**社内向けの背景情報は含めない**。
- 不要な「申し訳ありません」は入れない。本当に負担をかけた相手にだけ、1文で簡潔に。
- 認識ズレが発生した場合の経緯説明は、過剰に詳細を書かず、現実の運用フロー（「先方の通常フローで進んでしまって、〇〇の方との面談で調整になっておりました」など）の事実ベースで簡潔に。候補者にプレッシャーをかけたり期待値を狂わせる引用（「鈴木代表に直接お会いいただく前提」など）は引っ込める。

### 日程候補の提示
- 候補者の不可日程を共有するときは、「{日付}以降」と限定するより「上記を除いて」と幅広に伝える。早期実現を妨げない。
- HERP メールでも署名は省略可（過去のスレッド内で既に認知されているため）。

# 文体ガイド（共通）

佐藤 雄太の実際の Gmail / Craft / Slack のサンプルから抽出した文体ルール。
推薦文・スカウト文・クライアント連絡など、どの場面でも基本ルールはここに従う。

---

## 1. 基本トーン

- **ですます調・敬体**。
- 「優秀なビジネスマンとしての振る舞い」を意識。
- 角が立たない／ネガティブな印象にならない／過度にへりくだらない。
- 流れる文章を基本としつつ、要点は3つ前後の箇条書きで補強。
- 1文は簡潔に。ただし**文の途中（特に読点「、」）では改行しない**。改行は文末「。」や段落の区切りのみとし、一文は折り返さず書く（AIっぽい不自然な折り返しを避ける）。
- **数字にカンマ（,）を付けない**（2026-06 佐藤指示）。対外メッセージ（スカウト文・メール・メッセージ）で金額・数値を書くときは三桁区切りの「,」を使わず、「1200万円」「800万」のように書く。人が書くときに付けないため、付けるとAIっぽくなる。
- **迷ったらカジュアルよりプロフェッショナル寄り**に倒す。親しみ（相手の近況に触れる等）は保ちつつ、語彙と敬語はきちんとさせる。

## 3. 頻出フレーズ

### 書き出し
- 「お世話になっております。Pole&Lineの佐藤です。」
- 「はじめまして。Pole&Lineの佐藤と申します。」
- 「ご覧頂き有難うございます。」（スカウト返信時など）

### 連絡理由
- 「〜について、要件の確認でご連絡いたしました。」
- 「ご経歴を拝見し、ご活躍いただけるのではと思い、ご連絡差し上げました。」
- 「ぜひ一度お打ち合わせのお時間を頂戴したくご連絡しました。」

### 提案・推し
- 「〜という印象を持っております。」
- 「〜とお見受けしており、貴社にマッチするのではと考えております。」
- 「〜にご関心をお持ちです。」
- 「短いお時間でも結構ですので、一度オンラインで詳細をご説明できればと思います。」

### 結び
- 「どうぞよろしくお願いいたします。」
- 「引き続きよろしくお願いいたします。」
- 「ご検討いただけますと幸いです。」

### 候補者の転職理由を伝えるとき（必ずポジティブに）
- 「現職への不満はなく、〜のフェーズを志向。」
- 「より責任範囲が大きいところに挑戦したい」

## 5. やってはいけない表現

- ❌ 候補者の前職を否定的に書く（「組織が崩壊して」など）
  → ✅「組織の変化を経て、自身の経験が活きる環境を再考している」
- ❌ クライアントを上から目線で評価する
- ❌ 「絶対」「必ず」など断定的・過剰な表現
- ❌ カタカナ語の乱用（必要なものは使う）
- ❌ 候補者名を文字数カウントに含める（推薦文の300字制限など）
- ❌ 「〜という整理でした」（やや事務的・分析的すぎて佐藤の語り口に合わない）
  → ✅「〜のようです」「〜とのことでした」など、先方の判断としてやわらかく伝える
- ❌ くだけた語・俗語・横文字スラング（「座組」「ハマる」「温度感」「〜なやつ」「めっちゃ」「耳に入れておく」など）
  → ✅ プロフェッショナルな言い換え（「ご経験にフィットする」「ご関心の度合い」「〜のようなもの」「お知らせする」）
- ❌ 強調のための引用符（"ど真ん中すぎる" 等）や感嘆符（！）の多用＝AIっぽさの典型
  → ✅ 記号に頼らず言葉で表現する
- ❌ 体言止め・省略形のくだけた結び（「〜まで。」「〜かなと。」「〜だなと思って」）の多用
  → ✅ 文として締める（「〜と思い、ご連絡しました」「〜と考えております」）
- ❌ 口語の接続詞（「なので」「だから」「あと」）
  → ✅ 書き言葉（「そのため」「したがって」「また」）
- ❌ 尊敬語・謙譲語の欠落（「言っていた」「経験」「やられてきた」）
  → ✅「おっしゃっていた」「ご経験」「取り組んでこられた」「ご志向」

### AIっぽさを消す before → after（佐藤による実添削例）

Claude のドラフトを佐藤が手直しした際の典型パターン。今後はこの水準を初稿から目指す。

| Claude 初稿（AIっぽい／カジュアル） | 佐藤 修正（プロフェッショナル） |
|---|---|
| お疲れさまです、佐藤です。 | お世話になります。Pole&Lineの佐藤です。 |
| プロダクトマネージャーの募集が出ています。 | プロダクトマネージャーを採用されたいそうです。 |
| ビズリーチやenechainでやられてきたことそのもので、スキル的にはむしろ"ど真ん中すぎる"くらいハマる座組だなと思って、まずお知らせまで。 | エンタープライズ向けにユーザーの声を聞きながらプロダクトを伸ばしていく動きは酒見さんのご経験ど真ん中ではと思い、ご連絡しました。 |
| 前に言っていた | 前におっしゃっていた |
| なので"絶対これ！"というより、一応耳に入れておきますね、という温度感です。 | そのため、めちゃくちゃお勧め、というわけではございませんが、ご経験のフィット感からお知らせしました。 |
| 頭の片隅に置いてもらえればで大丈夫です。…ギフティのときみたいなやつが出てきたら | （冗長な気遣いの重ねは削る）…もっと合うものが出てきた際に、改めてご連絡しますね。 |
| 強く共感しています／非常に／もっとも近いと感じます | 共感しています／（副詞を落とす）／近いと感じます（強調副詞は初稿から抑える） |
| （箇条書き）データ起点の優位で重なります。◯◯を事業責任者として担う役割です。 | データ起点の優位で重なります。（見出しの「◯◯候補」で既出の役割は本文で言い直さない＝1項目1メッセージ） |
| 本日はお時間いただきありがとうございました（実際の送信は翌日） | 昨日はお時間いただきありがとうございました（日付語は `Today's date` と事象日で都度検算） |
| お世話になっております。Pole&Lineの佐藤です。（面識ある相手への継続連絡） | Pole&Lineの佐藤です。（既知の相手への継続連絡は冒頭挨拶を最小化） |

**学び：** ①書き出しは社名込みの定型で丁寧に ②尊敬語・謙譲語を徹底（ご経験／おっしゃる／ご志向）③横文字スラング・引用符強調・体言止めを排す ④気遣いフレーズを重ねず簡潔に ⑤親しみ（相手の近況・固有のやり取りへの言及）は残す。⑥強調副詞（強く／非常に／もっとも）は抑える。⑦箇条書きは1項目1メッセージにし、見出しで述べた役割を本文で繰り返さない。⑧日付語（本日／昨日／先日）は送信日と事象日で検算する。⑨既知の相手への継続連絡は冒頭挨拶を最小化する（①の「丁寧な定型」は初回接触向け。継続連絡では「Pole&Lineの佐藤です。」程度に留める）。

### 過度な丁寧さ・時制依存を削る（クライアント連絡の実添削）

2026-06、BASE 成原さん宛の面談フォローメールで佐藤が手直しした例。Claude の初稿はやや over-formal だった。前項がカジュアルすぎを戒めるのに対し、ここは**逆方向（丁寧すぎ・硬すぎ）も削る**という学び。

| Claude 初稿 | 佐藤 修正 | 学び |
|---|---|---|
| BASE株式会社　成原様 | 成原様 | 既に同じスレッド内で社名が自明なら、宛名は `{姓}様` のみで十分。社名を重ねると硬い（自己紹介の「Pole&Lineの佐藤です」は残す） |
| 本日はお忙しい中、〜ありがとうございました | お忙しい中、〜ありがとうございました | 「本日は」等の時制ワードは、下書きの送信が翌日以降にずれると不整合になる。送信タイミングが確定しない文面では時制を固定しない（CLAUDE.md の日付・都度確認と整合） |
| 誠にありがとうございました | ありがとうございました | 「誠に」等の過剰な美化・へりくだりは削る。プロフェッショナル＝丁寧だが過度にへりくだらない |
| ご質問など出てまいりましたら私からも間に入って調整いたします | 必要に応じて私からも間に入って調整いたします | 条件節は冗長に説明せず簡潔に（「必要に応じて」で足りる） |

**学び：** 狙いは"自然で品のあるプロフェッショナル"。over-formal の典型（誠に／本日は／宛名の社名重ね／冗長な条件節）も削る。特に時制ワードは送信遅延で崩れるため、タイミングが確定しない下書きでは使わない。

## 6. フォーマット規約

- Craft に貼り付ける前提で、マークダウン記法で出力する。
- 見出しは `##` / `###` を用い、過度なネストは避ける。
- 箇条書きは `-` を使う。
- 強調は `**太字**` を控えめに。

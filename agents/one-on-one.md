# 1on1 報告フィード（代表＝斉藤さんとの1on1 準備）

代表（斉藤さん）との1on1で、アップデート・報告・相談をしやすくするための運用。候補者・クライアント・スカウトの状況を更新するたびに、Craft の**1on1報告フィード（正本）**へ「日付＋概要」を1行追記して蓄積し、1on1準備時はそこから前回以降のエントリを拾って報告・相談ネタを組み立てる。

> 目的は「全体把握」。普段の業務（プロファイル更新・スカウト対応）で得た動きを、その場で薄く1行残しておくことで、週次の1on1準備が"拾うだけ"になる。新しい貯め場所を増やすのではなく、**プロファイル md を更新する行為にフックして自動で流し込む**のが肝。

## 1. フィードのエントリ書式

```
YYYY-MM-DD｜{種別}｜{対象名}｜{1行概要}〔{タグ}〕
```

- **種別：** 候補者 ／ クライアント ／ ポジション ／ スカウト
- **タグ：** 🔴要相談（代表の判断・助言が欲しい）／ 🟡報告（共有したい）／ 🟢進捗（順調・参考）
- **対象名：** 候補者は `{姓} {名}様`、クライアント／ポジションは正式名。
- **1行概要：** 動きの要点＋（あれば）相談の論点を1文で。CLAUDE.md §4 の文体に合わせ、文の途中で改行しない。

例：

- `2026-06-11｜候補者｜小林 中様｜ギフティCISO候補で1次面接通過、2次は来週前半で調整中。年収レンジの開示範囲を相談したい 〔🔴要相談〕`
- `2026-06-11｜クライアント｜NodeX｜法務リーダーの要件が英語必須に変更。既存パイプラインの適合再確認が必要 〔🟡報告〕`
- `2026-06-10｜スカウト｜ギフティ役員直下v2｜今週返信3件・うち1名カジュアル設定。toC版は返信率2%で停止し配分を役員直下へ寄せた 〔🟢進捗〕`

## 2. 追記ルール（いつ・どう書くか）★フック

### いつ追記するか（「報告に値する動き」だけ）

候補者・クライアント・スカウトに**実質的な動き**があったときに1行足す。具体的には：

- 選考フェーズの進行（書類通過・面接通過・オファー・内定受諾 など）
- 面談・打ち合わせの実施（所感の要点）
- 推薦提出、新規打診の開始
- スカウトの返信・新規接点化、媒体/文面の停止・差し替え判断
- 要件・年収条件の変更、クライアント都合の変化
- 辞退・離脱・見送り、No Show など
- 代表に判断・助言を仰ぎたい事象（🔴）

**載せないもの：** 軽微な同期・表記修正・`Last synced` だけの更新など、報告価値のない差分。ノイズを増やすとフィードが死ぬ。

### どう追記するか

最新を上に積むため、**更新フィード見出しの直後に挿入**する：

```
blocks add --siblingId 61516f55-e337-6b29-fcf8-bdc2dc22d2e4 --position after \
  --markdown "- YYYY-MM-DD｜{種別}｜{対象名}｜{概要}〔{タグ}〕"
```

- 日付は CLAUDE.md §6 のとおり **JST の正本**（SessionStart フックの「日付の正本（JST）」）を使う。`Today's date`（UTC）を鵜呑みにしない。
- 🔴要相談は、フィードに積むのに加えて **pinned「📌 次回1on1で相談したいこと」** にも1行足しておくと、準備時に取りこぼさない。
- 1回の動きで複数対象に波及する場合は、対象ごとに分けて1行ずつ。

### プロファイル更新フックとの接続

- **候補者 md を更新したとき**（`agents/candidate-profile.md` の §5.3 対話追記 / §5.4 同期更新）：選考状況・面談メモ・推薦履歴に報告価値のある差分が出たら、その差分要約をユーザーに返すのと同時にフィードへ1行追記する。
- **クライアント／ポジション md を更新したとき**（`agents/client-profile.md` の対話追記 / 同期更新）：要件変更・パイプライン進捗・打ち合わせ所感など報告価値のある差分が出たら同様に追記する。
- **スカウト対応のとき**（`agents/scout.md` / `agents/scout-kit.md`）：返信・新規接点・PDCA での停止/差し替え判断が出たら追記する。

## 3. 1on1 準備フロー（フィードから拾う）

「1on1の準備」「斉藤さんとの1on1」「報告まとめて」等が言及されたら：

1. **前回1on1の日付を確定**：Notion 1on1 DB の最新ページ（または `斉藤さん` アーカイブの最新リンク）から取得。
2. **フィードを読む**：`1on1 報告フィード｜斉藤さん` の「🗒️ 更新フィード」と pinned「📌 相談したいこと」を `blocks get` で展開し、**前回1on1日付より後のエントリ**を抽出。
3. **ToDo を重ねる**：Notion ToDo DB（`collection://2257d017-b6a0-8026-867c-000bb0969507`）の `Ower / 1on1 = 1on1` かつ ステータス≠完了 を `tools/notion_active_todos.py`（§10）で取得し、フィードに無い相談事項を補う。
4. **前回宿題の進捗**：前回議事録（Notion 1on1 DB／`斉藤さん` アーカイブ）の宿題・ToDo の消化状況を確認。
5. **構成して提示**：🔴要相談 → 🟡報告 → 🟢進捗 の順にまとめ、CLAUDE.md §6 の対外メッセージ自己検証（懸念点・出典チェック）に準じて要相談項目の論点を明確化する。必要なら清書ドラフトを Notion の次回1on1ページ本体に書き込む（§9 の議事録正本に合わせる）。
6. **会話で完結する場合**は提示まで。ページ反映までやるかはユーザーに確認。

## 4. 1on1 後のハウスキーピング

1on1 が終わったら：

- フィードで**報告・相談し終えたエントリ**を「✅ 過去ログ」へ移す（`blocks move` または旧エントリを過去ログ末尾へ add し直し）。月ごとにトグルで畳んでよい。
- pinned「📌 相談したいこと」を**クリア**し、消化した相談は議事録（Notion 1on1 DB／`斉藤さん`）へ送る。
- 議事録側の更新は CLAUDE.md §9 の運用に従う（フィードは準備の足場、議事録が結論の正本）。

# クライアント向け文章エージェント

クライアント企業へ向けて書く文章を扱う。主に以下2種類：
- **推薦文／推薦状**：候補者を企業へ紹介する文書
- **クライアント連絡メール**：要件確認・打ち合わせ依頼・進捗報告・日程調整

---

## 推薦文

### 1. 全体構造（Craft上のフォーマット）

```
# 推薦状

***

### {クライアント企業名}御中

#### ご紹介ポジション

`{ポジション名}`



#### 担当

Pole&Line合同会社
佐藤 雄太
Mobile：080-3930-1714

+ ## 基本情報

  氏名：{候補者氏名（ふりがな）}

  年齢：{年齢}歳

+ ## 推薦理由

  {280-300文字の推薦本文}

+ ## ご転職理由

  {現職への不満ではなく、ポジティブな志向として整理した転職理由}
```

### 2. 推薦理由本文（核となる280-300字）

#### 構造の典型パターン
1. **貴社の事業理解**：「貴社は〇〇事業や〇〇に取り組まれている印象を持っております。」
2. **候補者の経験紹介**：「{姓}様は、〇〇でのご経験をお持ちです。」
3. **マッチング根拠**：「特に〇〇まで一気通貫でリードできる方とお見受けしており、貴社にマッチするのではと考えております。」
4. **貢献領域の箇条書き**（任意）：3点程度で「〇〇推進」「〇〇設計」など。

#### ルール
- **ですます調で提案するような文体**。
- 候補者の名前は文字数カウントに**含めない**（テンプレートで埋めるため）。
- 文字数は **280-300字程度** を厳守（指定がある場合はそれに従う）。
- 「貴社」「{姓}様」を一貫して使う。
- 「お見受けしており」「と考えております」「ご関心をお持ちです」など、断定を避けた提案調。

### 3. ご転職理由

- ネガティブな表現を排する。
- 「現職への不満はなく」を入れて、現職評価が低くないことを明示。
- 「次のステージ」「フェーズを志向」「より責任範囲が大きいところに挑戦したい」など前向きな言葉で。

### 4. 作成プロセス

1. **会社情報まとめ**（Gemini/Perplexity）：事業・業績・成長戦略・最新リリースを把握。
2. **求人票の構造化**（Gemini）：JDをマークダウン化。
3. **候補者情報まとめ**：面談メモ（NotebookLM経由）と職務経歴書を統合。
4. **親和性評価**：SABCDの5段階で評価軸ごとに採点（キャリア／カルチャー／転職軸）。
5. **推薦文生成**：上記を踏まえて280-300字。
6. **Craftで体裁を整える**。

### 5. 親和性評価フォーマット（任意の付属物）

推薦文に評価サマリを添える場合：

| 評価軸 | 評点 | コメント |
|---|---|---|
| キャリア／スキルの一致度 | S/A/B/C/D | 〇〇のご経験が〇〇業務と一致 |
| カルチャーフィット | S/A/B/C/D | 〇〇環境での実績が同社カルチャーに合致 |
| 転職軸との整合性 | S/A/B/C/D | 候補者の〇〇志向が貴社のフェーズと一致 |

## クライアント連絡

### 1. メールの基本構造

```
{姓}様

{初回:はじめまして。Pole&Lineの佐藤と申します。}
{継続:お世話になっております。Pole&Lineの佐藤です。}

{本文：要件・目的を1〜2文で}

{詳細：必要に応じて箇条書きや候補日程}

{結び：どうぞよろしくお願いいたします。 / 引き続きよろしくお願いいたします。}

--
Pole&Line合同会社
佐藤 雄太｜SATO Yuta
Mobile：080-3930-1714
```

### 2. シーン別の書き出し・結び

#### 打ち合わせ依頼
- 件名：`【お打ち合わせ依頼｜Pole&Line】{ポジション名}について`
- 書き出し：「{領域}を重点テーマとして継続的にサーチ・支援を行っております。」「募集要項および背景を拝見し、〜とのご認識を持ちました。ぜひ一度お打ち合わせのお時間を頂戴したくご連絡しました。」

#### 要件確認
- 件名：`【Pole&Line】{ポジション名}の要件確認につきまして`
- 書き出し：「求人票を拝見し、〜と理解しており、〜について確認させてください。」

#### 候補者推薦
- 件名：`【ご推薦｜Pole&Line】{ポジション名}｜{候補者氏名}様`
- 本文に推薦文ファイルへのリンクや添付を明示。

#### 日程調整
- 「{日付候補}でご都合いかがでしょうか。」
- URLでの日程調整ツール返答時：「{選択日時}で確保させていただきました。」

### 3. 守るべきルール

- 件名は `【Pole&Line】` プレフィックスを基本とする（社内識別のため）。
- 季節の挨拶（明けましておめでとうございます等）は時期に応じて入れる。
- 「むしろ本ポジションについては、〜ありがたく思っております。」のような積極的な姿勢を示す表現を活用。
- 「全く問題ございません」と一言添えて相手の負担を下げる。
- 返信が遅れた場合の謝罪は短く一文で。長々と謝らない。

### 4. 候補者の懸念・温度感の伝え方（面談後フィードバック時）

候補者の事業内容への懸念・ポジションへの違和感など、ネガティブに受け取られうる温度感をクライアントへ伝える際は、**タイミングを段階的に分ける**。

#### 出さないほうがよいケース

クライアント側のFB受領前、または候補者本人が面接で直接伝えていない懸念を、エージェントから先出ししない。理由は3つ：

1. クライアントの評価軸とズレた印象操作になりかねない
2. 「本人の温度感が低い」と一段ネガティブに受け取られる（実際は「マッチを感じて前向き」が本筋であっても）
3. 想定ポジション提示や次フェーズで自然に解消する可能性がある

#### 面談直後のFBで出すべき要素

- **ポジティブな所感**：面談者個人への印象、企業/事業へのフィット感
- **お詫び・補足**（必要時）：当日の段取り上の事象など、クライアント側も目撃しているもの
- **次フェーズの期待値**：「全体的に前向き」「次フェーズで具体的に擦り合わせたい」までに留める

#### 段階的開示

クライアントFB＋想定ポジション提示が出揃った段階で、本人スタンスを「擦り合わせ材料」として共有する。

例：「○○の役割に寄ったご提案であれば、本人としては□□の方が強みを発揮できるとの認識のようでした」

#### 例外：すぐ伝えてよい補足

「面接中の中断」「日程変更の事情」など、本人がクライアントに直接見せている事象についての補足は、むしろ早めに伝えるほうがフォローとして効果的。

### 5. 進捗報告のフォーマット

クライアント向けに「サーチ状況の中間報告」を送る場合：

```
{姓}様

お世話になっております。Pole&Lineの佐藤です。

{ポジション名}のサーチ進捗についてご報告いたします。

■ 現状
・接触候補者数：{n}名
・面談実施：{n}名
・推薦予定：{n}名

■ 所感
{ターゲット像との照合結果、ボトルネック、調整したい論点}

■ 次のアクション
・{アクション1}
・{アクション2}

引き続きよろしくお願いいたします。
```

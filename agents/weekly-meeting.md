# Weekly全体MTG 定例準備（フィードから拾う）

全社の週次「Weekly全体Meeting」に向けて、**前回定例〜今日**の佐藤の動きから、定例で共有・相談すべき項目を拾って議事録ドキュメントに置くための運用。**新しい貯め場所は作らず、既存の「1on1 報告フィード｜斉藤さん」を素材源として再利用**する（`agents/one-on-one.md` のフィードが正本）。

> 1on1フィードは「報告価値のある動き」を広めに蓄積している。定例はそれを**もっと絞ったバー**（全社の場・口頭で語る前提）で再フィルタするだけ。だから「フィードにはあるが定例には出さない」が普通に起きる。何もバーを越えなければ **「特記なし」**で良い。

## 2. 定例に出す3バケツとバー（2026-06 佐藤確定）

記載スコープ＝**前回定例〜今日にあったこと**の差分のみ。各項目は**論点が立つ程度の箇条**（整文しない。当日オンラインで繋ぎ、担当が自分の件を口頭で詳述する）。

| バケツ | 出すもの（バー） | 出さないもの |
|---|---|---|
| **①オファー周り** | オファー提示／受諾／辞退／内定など**選考の山場**の動き | 書類通過・面接調整中など途中経過（選考中は出さない） |
| **②クライアント打ち合わせ** | **佐藤自身が同席した**打合せの要点 | 同席していない打合せ（情報を持っていない＝出さない）、定例より後の予定 |
| **③ナレッジ／相談** | 今**出す価値がある**相談（積極的に動く候補者の出し先相談 等）・**決定がまとまってからの**ナレッジ共有 | 本人が動かない候補者の相談、まだ実績の少ない運用ナレッジ（決定が積み上がってから） |

→ 3バケツのいずれも越えなければ **「特記なし」**と明示してよい（毎週ネタがあるとは限らない）。

## 3. 準備フロー（「定例準備して」「定例用にフィードから拾って」等で起動）

1. **期間を確定**：前回定例日（カレンダー `Weekly全体Meeting` の直近過去回＝通常は前週。迷えば議事録docの最新日付ヘッダ）〜今日。
2. **フィードを読む**：`1on1 報告フィード｜斉藤さん`（§0 rootBlockId）を `blocks get` で展開し、**前回定例より後**の佐藤エントリを抽出。種別→バケツの当たりは下記。
   - フィード `候補者` × オファー/受諾/辞退/内定 → **①**
   - フィード `クライアント`/`候補者` × 打合せ実施（**佐藤同席のものだけ**） → **②**
   - フィード `🔴要相談` 等で**今出す価値のあるもの** → **③**
3. **定例バーで再フィルタ**（§2）：1on1向けに積んだ項目から、定例に出すものだけ残す。同席なしの打合せ・動かない候補者の相談・未成熟ナレッジは**落とす**。
4. **不足を補う**（任意）：パイプラインで今週オファー段階に動いた佐藤案件が漏れていないか、必要なら Notion で確認。フィード未記載の確定オファーがあれば拾う。
5. **3バケツの箇条で提示**：①②③の順、論点が立つ短文。CLAUDE.md §6 の秘匿配慮（年収・競合オファー等は論点だけ・外に出さない）に従う。**整文しない**。
6. **書き込み**はユーザー確認後：議事録docの最新ブロックの `■今週のオファー/決定報告`／`■アジェンダ` に佐藤名で追記（§4）。会話で完結なら提示まで。

## 4. 書き込み（Googleドキュメント）

- 最新の `*YYYY/M/D*` ブロックが既にあるか確認（無ければ司会が作る。佐藤が司会回でなければ通常は誰かが作成済）。
- `■今週のオファー/決定報告` … `Yuta Sato {クライアント} {ポジション} {候補者}・{オファー/受諾/辞退と要点}`。
- `■アジェンダ` … 相談・共有したい議題を1行（語る用の見出し）。
- ※全文読み込みは避ける。書き込み箇所の近傍だけ取得して追記する。

## 5. 司会が佐藤の回のとき（ローテ）

司会順＝**熊倉→櫻井→河内→佐藤→清水→古市→松村→山田→伊藤**。佐藤が司会の回は、議事録運用ルールに従い追加で：会議開始時に「自動メモ生成」を押す → 終了後に届く文字起こしを notebooklm（`4d5ff301-a385-4800-bf26-7bf49f18362a`）に登録 → ブリーフィングを議事録へ反映。佐藤の報告準備（§3）はこれと別に行う。

## 6. 注意

- 議事録docは巨大。**全文をコンテキストに通さない**（構造は本書 §1 が正本）。
- フィードと議事録は別物：フィードは準備の足場（Craft正本）、定例の結論は議事録doc。1on1のハウスキーピング（`agents/one-on-one.md` §4）と二重処理しないよう、定例で消化した項目はフィード上で扱いを区別する（1on1で再度出す価値がなければ過去ログへ）。
- 本書を改訂したら CLAUDE.md §12 に従い **main に反映**する。

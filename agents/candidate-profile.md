# 候補者プロファイルエージェント

候補者ごとに `candidates/{slug}.md` を作り、Notion / Salesforce / Gmail / Slack に散在する情報を1ファイルに集約・蓄積するためのガイド。

`/candidate {姓}` または `/candidate {姓} {操作}` で起動される。

---

## 0. 重要：プライバシー

- `candidates/` ディレクトリは `.gitignore` で除外されている。**コミットしない**。
- 候補者の同意なく他社へ展開されない情報（職務経歴書添付、年収詳細など）を扱うため、対外共有時は必ず本人確認。
- 候補者宛メールに「他社展開不可」等の指示がある場合、md の冒頭「留意事項」に必ず明記する。

---

## 1. 起動モード

ユーザーの発話と `candidates/{slug}.md` の存在有無で、以下3モードを自動判定する：

| モード | 判定条件 | 振る舞い |
|---|---|---|
| **初回生成** | ファイルが存在しない | Notion + SF + Gmail + Slack から横断収集して新規作成 |
| **対話追記** | ユーザーが特定の素材（メール本文・Slack 抜粋・面談メモなど）を貼り付けた | 該当セクションに追記・要約 |
| **同期更新** | 「最新に同期して」「選考状況を反映して」等の指示 | Notion・SF を再取得して該当セクションを差し替え／追加 |

不明な場合はユーザーに確認する。

---

## 2. ファイル命名規則

- パス: `candidates/{姓ローマ字小文字}-{名ローマ字小文字}.md`
  - 例: `candidates/tobimoto-masaaki.md`、`candidates/sato-yuta.md`
- 同姓同名がいる場合、サフィックスに会社名を入れる（例: `tobimoto-masaaki-mixi.md`）
- ローマ字化はヘボン式準拠。`姓（ふりがな）` / `名（ふりがな）` を Notion から取得して使う。

## 4. 候補者 md のテンプレート

新規生成時は以下のフレームに従う。情報がない項目は `（未取得）` と書いて空にせず残す（後で埋められるように）。

> **Last synced:** {YYYY-MM-DD HH:MM} (JST)

> **Notion:** {URL}
> **Salesforce:** {URL}

## 5. 各モードの詳細手順

### 5.1 初回生成

1. ユーザーから `{姓}` または `{姓 名}` を受け取る
2. **Notion 候補者DB を search**：`姓` でヒットを探す
3. ヒットが0件 → ユーザーに確認（Notion未登録？スペル違い？）
4. ヒットが複数 → 候補をリスト表示してユーザー選択（社名等で絞り込み）
5. 候補者ページを fetch → 基本プロフィール埋め
6. Notion パイプライン / 面談メモ / 選考評価 を `候補者` リレーション経由で取得
7. Salesforce: 候補者ページにある `SalesForce` URL があれば fetch、なければ氏名で salesforce_search_all
8. Gmail: 候補者の Email アドレスで `search_threads`（最大20件）。各スレッドを1行に要約
9. Slack: `slack_search_public_and_private` で `{姓}` を検索（最大10件、直近6ヶ月）。要約
10. テンプレに埋めて `candidates/{slug}.md` を作成
11. ユーザーに作成完了とパス、Sources の取得結果を報告

**取得が一部失敗してもファイルは作る**。失敗箇所は `（取得失敗：{理由}）` と明記。

### 5.2 対話追記

ユーザーが「以下のメールを記録」「Slackこれ追加」のように素材を貼り付けたら：

1. 候補者 md がカレントセッションで開いていれば対象 → なければ確認
2. 素材の種類を判定（Gmail / Slack / 面談メモ / Notion / その他）
3. 該当セクションに追記：
   - メール → セクション5に1行要約 + セクション8に原文
   - Slack → セクション6に要約 + セクション8に原文
   - 面談メモ → セクション3に構造化（転職軸／推し所／注意点）して追加
   - 推薦の打診結果など → セクション4を更新
4. `Last synced` のタイムスタンプを「最終更新」として更新（Sources は変えない）
5. 追記した内容のサマリをユーザーに報告

### 5.3 同期更新

「最新に同期」「選考状況反映」等の指示で：

1. Notion パイプライン / 選考評価 / 面談メモを再取得
2. セクション2（選考中ポジション）、3（面談メモ）、4（推薦履歴）を差し替え or 追加
3. 差分を要約してユーザーに報告（「面談メモ1件追加、選考ステータス2件更新」など）
4. Last synced 更新、Sources も更新

---

## 6. 出力時のチェックリスト

- [ ] `candidates/` ディレクトリが存在しなければ `mkdir -p` で作成
- [ ] ファイル名は ヘボン式ローマ字小文字 + ハイフン区切り
- [ ] 最上部の Last synced / Sources / Notion URL / SF URL が埋まっている
- [ ] 留意事項セクションが先頭にある（個別配慮事項がなければ「特記事項なし」と明記）
- [ ] 取得失敗箇所は隠さず明記
- [ ] 個人情報（電話番号・年収・職務経歴）が含まれることを認識し、`git status` でステージングされないことを確認

---

## 7. NG

- `candidates/` を git にコミットしない（`.gitignore` で除外済み）
- 候補者本人の同意なく外部（クライアント他、別候補者など）へ md の内容を共有しない
- 「他社展開不可」の指示がある場合、推薦先選定時にも遵守
- 推測で項目を埋めない（年齢・年収などは Notion / 本人発言の事実ベース）

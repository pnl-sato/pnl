# 評価フレーム｜法務（リーダー）｜株式会社ギフティ #GFT

※Claude.ai 外への共有禁止。
未接触候補のスカウト適性を5軸で構造化評価する。`要件.md` `経営陣スタンス.md` `過去推薦パターン.md` を併読すること。

---

## 0. 評価の前提

- 対象：BizReach 等で発見した**未接触候補**（Craft/SF に履歴なし）
- 評価対象材料：候補者プロフィール（職歴・年齢・年収希望・現職・志向の手がかり）
- 既接点候補（Craft に candidate md あり）はこのキットを使わず、Claude Code から直接 profile md を読んで評価する
- スコアは1〜5の5段階。3未満が1つでもある軸は「進めるべきか要再考」のシグナル

---

## 1. 軸①｜要件適合（重み：高）

### 採点基準

| 点 | 状態 |
| --- | --- |
| 5 | Must 全て満たし、Want 3つ以上該当。主担当経験を年数より具体性で語れる |
| 4 | Must 全て満たし、Want 1〜2つ該当 |
| 3 | Must の一部（経験年数 or 主担当領域のいずれか）にグレーゾーン |
| 2 | Must の半分以下、または企業法務以外（事務所・知財専業など）寄り |
| 1 | 法務経験そのものが浅い／関連性が薄い |

### チェック項目

- 企業法務実務経験 **3年以上**（主担当として）
- 民法／会社法／個人情報保護法／景表法 の相談対応経験
- ToC／決済領域の契約書・利用規約レビュー経験
- 他部門連携の新規サービス法務サポート経験
- 知財管理／コンプライアンス体制設計／訴訟係争／M&A／海外法務／IPO の歓迎要件

---

## 2. 軸②｜カルチャー適合（重み：最高・脱落要因の最頻軸）

### 採点基準

| 点 | 状態 |
| --- | --- |
| 5 | 当事者型・社会貢献志向・「物事の間に落ちる」テーマへの感度が明確。少人数精鋭フェーズの経験あり |
| 4 | 当事者型スタンスを示唆する経歴あり。コンサル／大手出身でも「整っていない領域に飛び込む」動機が読める |
| 3 | 中立。経歴からカルチャー適合の判断材料不足 |
| 2 | 大手安定企業出身でステップアップ志向の弱さが見える／専門特化型寄り |
| 1 | 戦略コンサル・IBD出身でタイトル志向／ギラギラ野心型／「制度が整った環境」を強く希望 |

### チェック項目

- **当事者型スタンス**：守るだけでなく事業の当事者として法務キャリアを築く志向
- **少人数精鋭適性**：一人法務／組織化フェーズの経験、または手挙げ志向
- **「運縁勘」許容度**：「正しく判断する」より「なんとなくこっち」を受け入れられるか
- **コミュニケーション**：冗談を言える・距離感を縮められる温度感
- **ギラギラ／タイトル志向**：ギフティとミスマッチ
- **コンサル／IBD出身**：基本フィット難前提。例外を見出す観点で評価

---

## 3. 軸③｜年収適合（重み：高）

### 採点基準

| 点 | 状態 |
| --- | --- |
| 5 | 現職年収・希望年収ともに 750〜900万円のストライクゾーン |
| 4 | 700〜750万 または 900〜1,000万。SO込みで折り合いの余地大 |
| 3 | 650〜700万（少しストレッチ要）または 1,000〜1,100万（実勢を超えるが交渉余地） |
| 2 | 1,100〜1,300万（数百万差でミスマッチが頻発するゾーン） |
| 1 | 1,300万円超または500万円台以下 |

### チェック項目

- 公表レンジ 700〜1,000万円＋SO
- 実勢 800〜900万円（マネージャー期待値で900近く）
- メンバー枠（〜715万＋SO）との差別化軸
- 「数百万差で折り合わず」が頻発する 1,000万後半ゾーンは要注意

---

## 4. 軸④｜フェーズ適合（重み：中〜高）

### 採点基準

| 点 | 状態 |
| --- | --- |
| 5 | HD化・M&A・海外展開のフェーズに「自分が貢献できる絵」を描ける経歴（M&A法務／グループガバナンス／海外法務 の経験） |
| 4 | プライム上場企業／成長期 SaaS の法務経験あり、フェーズ感の理解が想像できる |
| 3 | 大手安定環境のみ／逆に超アーリーシード経験のみで、ギフティの「整いつつあるが拡大局面」へのフィット要確認 |
| 2 | フェーズの読み違いリスクあり（規模違いすぎる） |
| 1 | 公務員・士業など、事業会社経験そのものがない |

### チェック項目

- 上場前後／グループ再編／M&A 加速期 の経験
- 国内 → 海外展開フェーズの法務経験
- 「整っていない領域に飛び込む」経験の有無
- 規模変化（一人法務 → チーム化）の経験

---

## 5. 軸⑤｜過去パターン適合（重み：中）

### 採点基準

| 点 | 状態 |
| --- | --- |
| 5 | 過去ギフティで通った人の共通点に複数該当（事業会社・若手抜擢・横断PJ・分析型） |
| 4 | 共通点に1〜2項目該当 |
| 3 | 中立。明確な該当も該当外もなし |
| 2 | 見送りパターンに1項目該当（例：大手安定出身、専門特化型） |
| 1 | 見送りパターンに2項目以上該当（例：コンサル出身＋報酬1,000万後半希望） |

### チェック項目

- ★ 通った人の共通点
  - 事業会社経営層経験者・若手抜擢
  - 部署横断PJ推進経験
  - 事業構造を深く理解する分析型
  - 他社オファー多い候補者（市場価値の客観指標）
- ★ 見送りパターン
  - 「会社名だけ知りたい」型 No Show 系
  - 大手安定企業出身・ステップアップ志向弱
  - 戦略コンサル・IBD出身（タイトル持ち30〜40代）
  - 報酬1,000万後半希望
  - 専門特化型
  - 完全マネジメント志向

## 6. 出力フォーマット指定（Claude.ai 側で生成）

候補者プロフィールを貼られたら、以下の構成で出力すること。

### 6.1 評価サマリ（200字以内）

候補者の特徴を1段落で要約。強調点・主要懸念・推薦可否の方向感を含む。

### 6.2 5軸スコア表

```
| 軸 | スコア | 一言 |
| --- | --- | --- |
| ① 要件適合 | x/5 | ... |
| ② カルチャー適合 | x/5 | ... |
| ③ 年収適合 | x/5 | ... |
| ④ フェーズ適合 | x/5 | ... |
| ⑤ 過去パターン適合 | x/5 | ... |
| **総合** | **xx/25** | ... |
```

### 6.3 強調点（3項目以内）

スカウト文で打ち出すべき訴求軸。候補者の経歴と本ポジションの接続点。

### 6.4 懸念点（3項目以内）

- 推薦時に確認すべきリスク
- 候補者側の温度感確認が必要なポイント
- 報酬ミスマッチの可能性

### 6.5 推奨アクション

- **A. 即時スカウト送信**（総合18点以上 ＋ 軸②カルチャーが4以上）
- **B. スカウト送信、ただし訴求軸調整**（総合15〜17点）
- **C. 保留・追加情報収集**（総合12〜14点、または特定軸が極端に低い）
- **D. 推薦見送り**（総合12点未満、または NG パターン明確該当）

### 6.6 末尾の追記用1行ブロック（必ず出力）

`評価履歴.md` の末尾にコピペするためのテーブル行：

```
YYYY-MM-DD｜イニシャル｜現職カテゴリ｜総合点｜判定（A/B/C/D）｜一言所感
```

例：
```
2026-05-30｜T.K.｜SaaS事業会社・法務リーダー｜21/25｜A｜決済領域経験＋少人数組織化経験で適合度高、報酬希望860万でストライク
```

## 7. 評価時の運用メモ

### 評価軸の重み付け

- ② カルチャー適合 が **最も脱落要因**になりやすい軸。経歴のキラキラ度より優先
- ③ 年収適合 は「実勢800〜900万」ストライクゾーンが厚い。1,000万後半希望者は **C 保留** が安全
- ① 要件適合 は3点以上を確保したい（3年実務未満は **D 見送り**）
- ④ フェーズ適合 は HD化・M&A・海外展開 のキーワードでフィット感を測る

### 不明情報の扱い

- プロフィールから読み取れない軸は「3点（中立）」とし、所感に「[要確認：◯◯]」を明記
- **推測で5点や1点を付けない**

### 進行中候補者の取り扱い

- 候補者から返信が来た時点で Claude Code 側に candidate md を新規作成し、以降は正本連動運用へ
- Claude.ai プロジェクトは未接触プール専用と割り切る

### 月次リセット

- `評価履歴.md` を Claude Code 経由で `ギフティ/positions/法務（リーダー）/scout_log_{YYYY-MM}.md` に書き出す
- プロジェクト側の `評価履歴.md` を空テンプレに戻して再保存

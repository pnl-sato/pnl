## 0. このプロジェクトの問い と 結論

問い：「いまの Claude Code の使い方から、月収100万円を得る方法を分析評価する」。

2つの道に整理し、佐藤さんが **(Y) を選択**：

- **(X) 別収入（副業）案** … Claude Code の構築スキル自体を研修/コンサル/コンテンツで売る。→ **却下**（評価記録は §0.5）。
- **(Y) 本業（採用）の数字を AI で取り戻す** … ★採用。確率が高く競業リスクゼロ。佐藤さん自身の過去データが「到達可能」を実証。

### 0.5. 却下した (X) 副業案の評価記録（復活検討時の参照用）

前提条件の変遷：当初「本業とは別の収入源」として相談 → 「本業は続けながら」と確定 → 「厳しく見て100万いくか?」に対し**低確率と評価** → (Y)へ転換。

5経路の評価（本業継続・時間制約下）：

| 経路 | 100万到達現実性 | 主な難点 |
|---|---|---|
| A. AI業務導入コンサル/受託 | 単独では高いが | 労働集約で「第二の本業」化、競業避止に抵触しやすい |
| B. 企業研修/講演 | 中〜高 | 月5本は本業持ちには夜が潰れる |
| C. 発信→教材/サロン | 単独では低 | 100万まで1〜2年、ネタの賞味期限が先に切れる |
| D. テンプレ販売 | 低（補助） | 集客が課題 |
| E. 紹介手数料 | 低（余録） | 不安定 |

却下の核心：**「100万に届く版(高単価B2B)は本業継続と衝突し、本業と両立する版(非同期発信)は100万に届きにくい」という構造的矛盾**。加えて本業の数字が落ちている状況で副業に時間を割くと、代表からのリモート継続条件（数字を上げる）を失うリスク。正直な天井は本業継続下で「6〜12ヶ月で月10〜30万」が現実的と評価。
→ 同じ Claude Code の力を**本業に向ける方が、月100万への確率が高い**と判断し (Y) へ。

## 2. 診断と100万への数式

漏れはAIの効く順に：
1. **🔴 返信率 5.17%**（ファネル最大・最広・最もAI向き）。5%→8-10%に上げれば下半分すべてが比例で太る。改善手段＝スカウト文の刺さり・個別化・媒体選択＝`scout.md`/`scout-kit.md`が直撃。**最優先。**
2. **🟡 書類通過 47.8%**（マッチング精度）。`client-profile.md §7` の5軸マッチングをアプライ前に必須化。
3. **🟠 オファー転換17%＋辞退最多**（候補者の動機づけ・経営陣の買い込み）。AIは準備まで、足を運ぶのは佐藤さんの仕事。
4. **戦略軸：マイクライアント集中**（フィー単価最大化の掛け算）。

**数式：** 月収100万＝年1,200万（取り分35%仮置きで年billings約3,400万）＝**決定 年6件**が必要。直近run-rate 年3件 → **2倍**。2022–23に達成済みの数字。ざっくり「約750〜1,200送信で1決定」。返信率の倍化＋書類通過の改善＋本丸集中の掛け合わせで到達を狙う。
